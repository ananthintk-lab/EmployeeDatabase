## Codebase Patterns
- `InMemoryEmployeeRepository.save()` auto-generates an id via an `AtomicLong idSequence` only when `employee.getId() == null`. When honoring a client-supplied id, advance the sequence past it (`idSequence.updateAndGet(current -> Math.max(current, employee.getId()))`) to prevent future auto-generated ids from colliding with it.
- Conflict-style exceptions (duplicate email, duplicate id, etc.) each get their own `RuntimeException` subclass in `exception/` and a matching `@ExceptionHandler` method in `GlobalExceptionHandler` that delegates to the shared `buildResponse(HttpStatus, message, request)` helper — follow this pattern for new conflict cases instead of reusing an existing exception type.
- `pom.xml` and `mvnw`/`mvnw.cmd` live in the `EmployeeDatabase` root, one level above this `defect` directory — run Maven commands from there (e.g. `..\mvnw.cmd verify -q` or `cd` up first), not from `defect`.
